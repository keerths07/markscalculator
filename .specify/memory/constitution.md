\# Project Constitution



\## Core Principles

\- Write clean, readable Python code

\- Always include tests for new features

\- Never commit secrets or sensitive data

\- Ensure CI passes before merging

\- Follow PEP8 standards



\## Security Rules

\- Run Bandit for security scanning

\- Run dependency audit before release

\- No unsafe dependencies allowed



\## Quality Rules

\- Maintain linting with Ruff, Flake8, Pylint

\- Maintain type safety with MyPy

\- Maintain test coverage above threshold
